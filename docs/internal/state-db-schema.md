# `state.db` schema

`state.db` is the SQLite database that backs agent-deck's per-profile session,
group, watcher, and cost-tracking state. It replaced the legacy
`sessions.json` file in v0.11.0 and is the canonical persistence layer for
everything the TUI and CLI need to reload across restarts.

This document describes the on-disk schema as of the schema version recorded
in the `metadata` table. It is generated by hand from the migration source in
`internal/statedb/statedb.go` and verified against `sqlite3 state.db .schema`
on a freshly initialised profile.

## Location

```
~/.agent-deck/profiles/<profile>/state.db
```

Resolved in `internal/session/config.go:GetProfileDir`. Each profile (e.g.
`personal`, `work`, `test`) gets its own `state.db`; profiles are otherwise
isolated. Alongside `state.db` SQLite maintains the usual WAL companions
`state.db-wal` and `state.db-shm`.

## Connection setup

`statedb.Open` (`internal/statedb/statedb.go`) opens the file with these
per-connection pragmas, set via the DSN's `_pragma` parameter so they apply
to every pooled connection rather than just the first one to run `PRAGMA`:

- `journal_mode=WAL` — multi-process readers + one writer without blocking
- `busy_timeout=5000` — wait up to 5s on a contended writer slot
- `foreign_keys=ON` — enforce the FK from `watcher_events.watcher_id`

Short-lived writes also go through `withBusyRetry` (linear backoff: 10, 20,
30, 40, 50 ms; ~150 ms total) as a belt-and-suspenders fallback when the
driver still surfaces `SQLITE_BUSY` at the application level under heavy
multi-writer load.

## Schema version

The current schema version is **`7`** (the `SchemaVersion` constant in
`internal/statedb/statedb.go`). It is recorded as a row in `metadata`:

```
key='schema_version', value='7'
```

Migrations run inside a single transaction on every `Open`. They are
idempotent in two ways:

1. All `CREATE TABLE` statements use `IF NOT EXISTS`.
2. All `ALTER TABLE … ADD COLUMN` statements catch the SQLite
   `"duplicate column"` error and swallow it.

Two migration paths coexist:

- An always-run `alterMigrations` slice that re-asserts every column added
  after the original schema. This handles databases that predate the
  numbered schema-version scheme.
- A numbered ladder (`oldVer < 4`, `oldVer < 6`, `oldVer < 8`) that bumps
  `metadata.schema_version` and runs any version-specific backfills (e.g.
  setting `is_conductor = 1` for sessions whose title starts with
  `conductor-`).

Note: the `SchemaVersion` constant is `7`, but the migration code already
contains a `oldVer < 8` branch that ALTERs `instances.title_locked`. The
column is also re-asserted unconditionally via `alterMigrations`, so a
freshly initialised DB always has it regardless of the version constant.

### `metadata`

Generic key/value store. Currently used only for the schema version, but
intentionally generic so future migrations can stash settings without
needing yet another table.

```sql
CREATE TABLE metadata (
    key   TEXT PRIMARY KEY,
    value TEXT NOT NULL
);
```

| Column | Type | Constraints | Meaning | Example |
|---|---|---|---|---|
| `key` | TEXT | PRIMARY KEY | Setting name | `schema_version` |
| `value` | TEXT | NOT NULL | Stringified value (no typed value column on purpose) | `7` |

Go mapping: no dedicated struct. Read/written via `StateDB.GetMetadata` /
`SetMetadata` (string keys/values).

### `instances`

One row per session managed by agent-deck. The TUI and CLI both consider
this the source of truth — `internal/session/storage.go` loads it into
`InstanceData` structs which are then hydrated into the live
`session.Instance` runtime type.

```sql
CREATE TABLE instances (
    id                   TEXT PRIMARY KEY,
    title                TEXT NOT NULL,
    project_path         TEXT NOT NULL,
    group_path           TEXT NOT NULL DEFAULT 'my-sessions',
    sort_order           INTEGER NOT NULL DEFAULT 0,
    command              TEXT NOT NULL DEFAULT '',
    wrapper              TEXT NOT NULL DEFAULT '',
    tool                 TEXT NOT NULL DEFAULT 'shell',
    status               TEXT NOT NULL DEFAULT 'error',
    tmux_session         TEXT NOT NULL DEFAULT '',
    tmux_socket_name     TEXT NOT NULL DEFAULT '',
    created_at           INTEGER NOT NULL,
    last_accessed        INTEGER NOT NULL DEFAULT 0,
    parent_session_id    TEXT NOT NULL DEFAULT '',
    is_conductor         INTEGER NOT NULL DEFAULT 0,
    no_transition_notify INTEGER NOT NULL DEFAULT 0,
    title_locked         INTEGER NOT NULL DEFAULT 0,
    worktree_path        TEXT NOT NULL DEFAULT '',
    worktree_repo        TEXT NOT NULL DEFAULT '',
    worktree_branch      TEXT NOT NULL DEFAULT '',
    tool_data            TEXT NOT NULL DEFAULT '{}',
    acknowledged         INTEGER NOT NULL DEFAULT 0
);
```

| Column | Type | Constraints | Meaning | Example |
|---|---|---|---|---|
| `id` | TEXT | PRIMARY KEY | UUID-ish session ID, generated at create time | `e88abcdb-1773-...` |
| `title` | TEXT | NOT NULL | Display title shown in the TUI | `conductor-innotrade` |
| `project_path` | TEXT | NOT NULL | Working directory the session was launched in | `/home/user/repo` |
| `group_path` | TEXT | NOT NULL, default `my-sessions` | Group this session belongs to (joins to `groups.path`) | `conductor` |
| `sort_order` | INTEGER | default 0 | Display order within the group; lower = earlier | `10` |
| `command` | TEXT | default `''` | Tool executable (`claude`, `gemini`, `codex`, `shell`, …) | `claude` |
| `wrapper` | TEXT | default `''` | Optional shell wrapper around `command` | `direnv exec .` |
| `tool` | TEXT | default `shell` | Tool family (used for hook + sync routing) | `claude` |
| `status` | TEXT | default `error` | Last-known status string from `internal/sessionstatus` | `running`, `idle`, `working`, `error` |
| `tmux_session` | TEXT | default `''` | tmux session name (without socket selector) | `agentdeck_proj_e88abcdb` |
| `tmux_socket_name` | TEXT | default `''` | tmux `-L` selector (v1.7.50+, issue #687); empty = default server | `agentdeck` |
| `created_at` | INTEGER | NOT NULL | Creation time, Unix seconds | `1764512345` |
| `last_accessed` | INTEGER | default 0 | Last focus time, Unix seconds (0 = never) | `1764598765` |
| `parent_session_id` | TEXT | default `''` | If set, this is a sub-session of `instances.id = parent_session_id` | `e88abcdb-...` |
| `is_conductor` | INTEGER | default 0 | Boolean (0/1). v4 migration backfilled `title LIKE 'conductor-%'` | `1` |
| `no_transition_notify` | INTEGER | default 0 | Boolean. Suppress transition-event dispatch for this row (v6) | `0` |
| `title_locked` | INTEGER | default 0 | Boolean. Block Claude session-name sync into `title` (v8 / v1.7.52, issue #697) | `0` |
| `worktree_path` | TEXT | default `''` | Path to the git worktree this session owns (if any) | `/tmp/wt-xxx` |
| `worktree_repo` | TEXT | default `''` | Repo root the worktree was cut from | `/home/user/repo` |
| `worktree_branch` | TEXT | default `''` | Branch name inside the worktree | `feat/xyz` |
| `tool_data` | TEXT | default `'{}'` | JSON blob; see [tool_data shape](#tool_data-shape) below | `{...}` |
| `acknowledged` | INTEGER | default 0 | Boolean. UI ack flag for status transitions (v3) | `1` |

Boolean integers always use `0`/`1`; no `TINYINT`/`BOOLEAN` aliasing.

Time columns store **Unix seconds** as `INTEGER`. The Go marshalling layer
uses `time.Unix(secs, 0)` / `t.Unix()`.

Writes go through `SaveInstance` / `SaveInstances` (`statedb.go:496+`),
which both use `INSERT OR REPLACE INTO instances (...)`. `SaveInstances`
additionally deletes any row not in the provided list to keep the DB in
sync with in-memory state, and merges `tool_data` keys it doesn't model
(see [tool_data shape](#tool_data-shape)) so user-managed extras like
`clear_on_compact` survive a rewrite.

Go mapping:

- `internal/statedb`: `InstanceRow` (the SQL DTO)
- `internal/session`: `InstanceData` (JSON-tagged serialization type) →
  `Instance` (live runtime type)

#### `tool_data` shape

`tool_data` is a JSON blob, not a relational column, because the shape
varies per tool (claude/gemini/codex/opencode/…) and changes too often to
justify a column-per-field schema. The typed shape is defined by
`toolDataBlob` in `internal/statedb/migrate.go`. Known keys:

| Key | JSON type | Meaning |
|---|---|---|
| `claude_session_id` | string | Resume token for `claude --resume` |
| `claude_detected_at` | int (Unix s) | When the resume token was first observed |
| `gemini_session_id` | string | Gemini session token |
| `gemini_detected_at` | int (Unix s) | When the Gemini token was first observed |
| `gemini_yolo_mode` | bool (nullable) | Gemini YOLO toggle |
| `gemini_model` | string | Pinned Gemini model name |
| `opencode_session_id` | string | OpenCode session token |
| `opencode_detected_at` | int (Unix s) | When the OpenCode token was first observed |
| `codex_session_id` | string | Codex session token |
| `codex_detected_at` | int (Unix s) | When the Codex token was first observed |
| `latest_prompt` | string | Last user input shown in the TUI preview |
| `notes` | string | Free-form user notes |
| `loaded_mcp_names` | string[] | MCP servers attached at last spawn |
| `channels` | string[] | Plugin channels in effect (`--channels`) |
| `plugins` | string[] | Catalog-key plugin list (RFC `PLUGIN_ATTACH.md`) |
| `plugin_channel_link_disabled` | bool | Suppress auto-linking of plugin channels |
| `auto_linked_channels` | string[] | Channels added automatically — kept so reconciliation can clean them up later |
| `extra_args` | string[] | Extra CLI tokens appended to `claude` start/resume/fork |
| `tool_options` | object | Tool-specific launch options (free-form) |
| `sandbox` | object | Serialized `SandboxConfig` |
| `sandbox_container` | string | Docker container name backing the sandbox |
| `ssh_host` | string | SSH hop for remote sessions |
| `ssh_remote_path` | string | Remote working directory on `ssh_host` |
| `multi_repo_enabled` | bool | Multi-repo mode toggle |
| `additional_paths` | string[] | Extra repos opened in multi-repo mode |
| `multi_repo_temp_dir` | string | Scratch dir for multi-repo worktree mounts |
| `multi_repo_worktrees` | object[] | Per-additional-path worktree entries (`original_path`, `worktree_path`, `repo_root`, `branch`) |
| `color` | string | Per-session TUI row tint (issue #391) |
| `clear_on_compact` | bool | User-managed extra, preserved by the merge path but not modelled in `toolDataBlob` |

The merge logic lives in `internal/statedb/tool_data_extras.go`
(`MergeToolDataExtras`): on every `SaveInstance(s)`, the existing
`tool_data` row is read first, and any keys not present in the new typed
blob are overlaid before the write.

### `groups`

TUI groups — the buckets visible in the left rail of agent-deck. Joins to
`instances.group_path` by `groups.path`.

```sql
CREATE TABLE groups (
    path           TEXT PRIMARY KEY,
    name           TEXT NOT NULL,
    expanded       INTEGER NOT NULL DEFAULT 1,
    sort_order     INTEGER NOT NULL DEFAULT 0,
    default_path   TEXT NOT NULL DEFAULT '',
    max_concurrent INTEGER NOT NULL DEFAULT 0
);
```

| Column | Type | Constraints | Meaning | Example |
|---|---|---|---|---|
| `path` | TEXT | PRIMARY KEY | Stable group key (used in `instances.group_path`) | `conductor` |
| `name` | TEXT | NOT NULL | Human-readable label shown in the TUI | `Conductors` |
| `expanded` | INTEGER | default 1 | Boolean. UI fold state | `1` |
| `sort_order` | INTEGER | default 0 | Display order in the TUI rail | `5` |
| `default_path` | TEXT | default `''` | Default working directory for new sessions added to this group | `/home/user/work` |
| `max_concurrent` | INTEGER | default 0 | Cap on simultaneously running sessions in this group (v1.9.1). `0` = unlimited (legacy); `1` = serial (default for new groups); `N≥2` = bounded parallelism | `1` |

Writes go through `SaveGroups`, which does an `INSERT INTO groups (…)`
inside a transaction that first `DELETE`s rows not in the provided list,
so the table tracks in-memory group state exactly.

Go mapping: `statedb.GroupRow` ↔ `session.GroupData` ↔ `session.Group`.

### `instance_heartbeats`

Per-OS-process liveness records. Multiple agent-deck processes (CLI + TUI
+ subagent runners) write to the same `state.db` concurrently; this table
lets each process advertise that it's still alive, and the TUI uses
`is_primary` to elect a single owner for foreground work.

```sql
CREATE TABLE instance_heartbeats (
    pid        INTEGER PRIMARY KEY,
    started    INTEGER NOT NULL,
    heartbeat  INTEGER NOT NULL,
    is_primary INTEGER NOT NULL DEFAULT 0
);
```

| Column | Type | Constraints | Meaning | Example |
|---|---|---|---|---|
| `pid` | INTEGER | PRIMARY KEY | OS PID of the running agent-deck process | `48273` |
| `started` | INTEGER | NOT NULL | Process start time, Unix seconds | `1764500000` |
| `heartbeat` | INTEGER | NOT NULL | Last heartbeat time, Unix seconds | `1764500300` |
| `is_primary` | INTEGER | default 0 | Boolean. `1` for the elected primary; cleared when `heartbeat` ages past the staleness threshold | `1` |

The election logic (in `statedb.go`) demotes any primary whose `heartbeat`
is older than the staleness threshold, then promotes a fresh process.

Go mapping: no exported struct; helpers live on `*StateDB` (`Heartbeat`,
`ElectPrimary`, `ReleasePrimary`, etc.).

### `recent_sessions`

Tombstones for the "recently deleted sessions" picker. When a session is
deleted from `instances`, a row is written here so the user can re-create
it with the same config in one click. Capped at the **20 most recent**
deletions (older rows are pruned on every insert).

```sql
CREATE TABLE recent_sessions (
    id              TEXT PRIMARY KEY,
    title           TEXT NOT NULL,
    project_path    TEXT NOT NULL,
    group_path      TEXT NOT NULL DEFAULT '',
    command         TEXT NOT NULL DEFAULT '',
    wrapper         TEXT NOT NULL DEFAULT '',
    tool            TEXT NOT NULL DEFAULT '',
    tool_options    TEXT NOT NULL DEFAULT '{}',
    sandbox_enabled INTEGER NOT NULL DEFAULT 0,
    gemini_yolo     INTEGER,
    deleted_at      INTEGER NOT NULL
);
```

| Column | Type | Constraints | Meaning | Example |
|---|---|---|---|---|
| `id` | TEXT | PRIMARY KEY | SHA-256 dedup key over `title+project_path+tool+group_path` | `9f2a…` |
| `title` | TEXT | NOT NULL | Title of the deleted session | `feature-x` |
| `project_path` | TEXT | NOT NULL | Working directory of the deleted session | `/home/user/repo` |
| `group_path` | TEXT | default `''` | Group it lived in | `work` |
| `command` | TEXT | default `''` | Tool executable | `claude` |
| `wrapper` | TEXT | default `''` | Wrapper command | `''` |
| `tool` | TEXT | default `''` | Tool family | `claude` |
| `tool_options` | TEXT | default `'{}'` | Serialized `ToolOptionsWrapper` JSON | `{...}` |

### `cost_events`

Append-only Anthropic API cost ledger. The cost subsystem
(`internal/costs`) owns the canonical `CostEvent` type; the statedb
package keeps a thin DTO (`CostEventRow`) so it can read/write without a
circular import.

```sql
CREATE TABLE cost_events (
    id                    TEXT PRIMARY KEY,
    session_id            TEXT NOT NULL,
    timestamp             DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    model                 TEXT NOT NULL,
    input_tokens          INTEGER NOT NULL DEFAULT 0,
    output_tokens         INTEGER NOT NULL DEFAULT 0,
    cache_read_tokens     INTEGER NOT NULL DEFAULT 0,
    cache_write_tokens    INTEGER NOT NULL DEFAULT 0,
    cost_microdollars     INTEGER NOT NULL DEFAULT 0,
    budget_stop_triggered INTEGER NOT NULL DEFAULT 0
);
CREATE INDEX idx_cost_events_session   ON cost_events(session_id);
CREATE INDEX idx_cost_events_timestamp ON cost_events(timestamp);
```

| Column | Type | Constraints | Meaning | Example |
|---|---|---|---|---|
| `id` | TEXT | PRIMARY KEY | Event UUID (caller-supplied) | `f1a2…` |
| `session_id` | TEXT | NOT NULL | Foreign reference to `instances.id` (not declared as FK so cost history survives session deletion) | `e88abcdb-...` |
| `timestamp` | DATETIME | default `CURRENT_TIMESTAMP` | Event time, stored as **TEXT in RFC3339** by Go callers — preserved verbatim by the row scanner | `2026-05-16T08:00:00Z` |
| `model` | TEXT | NOT NULL | Model name, e.g. `claude-opus-4-7` | `claude-sonnet-4-6` |
| `input_tokens` | INTEGER | default 0 | Input tokens billed | `12000` |
| `output_tokens` | INTEGER | default 0 | Output tokens billed | `350` |
| `cache_read_tokens` | INTEGER | default 0 | Cached input tokens (read hits) | `9000` |
| `cache_write_tokens` | INTEGER | default 0 | Cache-write tokens | `12000` |
| `cost_microdollars` | INTEGER | default 0 | Cost in micro-dollars (1e-6 USD), integer for exact math | `42500` (= $0.0425) |
| `budget_stop_triggered` | INTEGER | default 0 | Boolean. Marks the event that crossed a configured budget threshold and triggered a stop | `0` |

The `timestamp` column has type affinity `DATETIME` but SQLite has no
native datetime; agent-deck writes RFC3339 strings, and `CostEventRow`
deliberately keeps it as a Go `string` to preserve the exact bytes.

Two indexes back the common queries: "all events for a session" and
"recent events across all sessions".

Go mapping: `statedb.CostEventRow` (thin DTO) ↔ `costs.CostEvent` (rich
type, defined in `internal/costs`).

### `watcher_events`

Per-watcher event log with **dedup + routing**. Every inbound event a
watcher sees is recorded here. `(watcher_id, dedup_key)` is unique, so
`INSERT OR IGNORE` is idempotent — a watcher can re-deliver the same event
on restart and we'll only keep one row.

```sql
CREATE TABLE watcher_events (
    id                INTEGER PRIMARY KEY AUTOINCREMENT,
    watcher_id        TEXT NOT NULL REFERENCES watchers(id),
    dedup_key         TEXT NOT NULL,
    sender            TEXT NOT NULL DEFAULT '',
    subject           TEXT NOT NULL DEFAULT '',
    routed_to         TEXT NOT NULL DEFAULT '',
    session_id        TEXT NOT NULL DEFAULT '',
    triage_session_id TEXT NOT NULL DEFAULT '',
    created_at        INTEGER NOT NULL,
    UNIQUE(watcher_id, dedup_key)
);
CREATE INDEX idx_watcher_events_watcher_created
    ON watcher_events(watcher_id, created_at DESC);
```

| Column | Type | Constraints | Meaning | Example |
|---|---|---|---|---|
| `id` | INTEGER | PK, AUTOINCREMENT | Monotonic event row ID (note: this is why `sqlite_sequence` exists) | `42` |
| `watcher_id` | TEXT | NOT NULL, REFERENCES `watchers(id)` | Owning watcher (FK enforced via `foreign_keys=ON`) | `9c5e562a-...` |
| `dedup_key` | TEXT | NOT NULL, UNIQUE with `watcher_id` | Watcher-defined idempotency key (e.g. Gmail message ID) | `198a2c1f...` |
| `sender` | TEXT | default `''` | Free-form sender (e.g. email From) | `alice@example.com` |
| `subject` | TEXT | default `''` | Free-form subject line | `Re: deploy` |
| `routed_to` | TEXT | default `''` | Display name of the session the event was routed to | `conductor-innotrade` |
| `session_id` | TEXT | default `''` | `instances.id` the event was ultimately delivered to | `e88abcdb-...` |
| `triage_session_id` | TEXT | default `''` | If a triage step ran first, the session that handled it (Phase 18 addition) | `f022320f-...` |
| `created_at` | INTEGER | NOT NULL | Unix-seconds insert time | `1764598765` |

`SaveWatcherEvent` does `INSERT OR IGNORE` and returns whether the row was
newly inserted, then prunes old rows down to a caller-supplied
`maxEvents`. `UpdateRoute` / `UpdateRouteWithTriage` patch the routing
columns post-hoc once the event has been delivered.

Go mapping: `statedb.WatcherEventRow`.

### `sqlite_sequence`

Auto-created by SQLite the first time an `AUTOINCREMENT` column is used —
in our case `watcher_events.id`. It stores one row per AUTOINCREMENT
table, recording the highest ID ever used. Do not modify directly.

## Migration history (summary)

The numbered ladder lives in `internal/statedb/statedb.go` around the
`existingVersion != schemaVersion` block.

| Version | Change |
|---|---|
| **v1** | Base schema: `metadata`, `instances`, `groups`, `instance_heartbeats` |
| **v2** | Added `recent_sessions` |
| **v3** | Added `cost_events` + its two indexes; added `instances.acknowledged` |
| **v4** | Added `instances.is_conductor`; backfilled rows whose `title LIKE 'conductor-%'` |
| **v5** | Added `watchers`, `watcher_events`, `idx_watcher_events_watcher_created`; later added `watcher_events.triage_session_id` |
| **v6** | Added `instances.no_transition_notify` |
| **v7** | Current `SchemaVersion`. Added `instances.tmux_socket_name` (issue #687, v1.7.50) and `groups.max_concurrent` (v1.9.1) |
| **v8** *(in-flight)* | Adds `instances.title_locked` (issue #697, v1.7.52). Re-asserted on every open via `alterMigrations` even when `metadata.schema_version` is still `7` |

Numbered version bumps are recorded by updating
`metadata.schema_version`. The always-run `alterMigrations` slice covers
the case of a database that pre-dates the numbered scheme, so any
combination of (old schema_version, current binary) converges to the
shape described above on the next `Open`.

## Verifying this document

After a schema change, regenerate a comparison dump from a fresh DB:

```bash
mkdir -p /tmp/sdb-check
HOME=/tmp/sdb-check ./agent-deck --profile check list >/dev/null
sqlite3 /tmp/sdb-check/.agent-deck/profiles/check/state.db .schema
sqlite3 /tmp/sdb-check/.agent-deck/profiles/check/state.db "SELECT * FROM metadata;"
```

Then diff against the `CREATE TABLE` blocks above.
